####### ELK Stack #######

ELK stands for Elasticsearch, Logstash, and Kibana—a powerful log management and analytics stack.
Elasticsearch stores the logs making search fast and scalable.
Logstash collects, processes, transforms, and ships logs from different sources into Elasticsearch.
Kibana visualizes the logs with dashboards, charts, and search capabilities.
ELK helps engineers troubleshoot, monitor applications, track errors, and analyze system behavior.
Widely used for centralized logging, observability, and real-time analytics.

To install a complete ELK stack (Elasticsearch + Logstash + Filebeat) on Docker Desktop Kubernetes, and then connect it to Grafana for log viewing.

1) Add Elastic Helm Repository
helm repo add elastic https://helm.elastic.co
helm repo update


2) Create a minimal values file:

$ cat es-values.yaml
replicas: 1

resources:
  requests:
    cpu: "500m"
    memory: "1Gi"
  limits:
    cpu: "1"
    memory: "2Gi"

volumeClaimTemplate:
  accessModes: [ "ReadWriteOnce" ]
  resources:
    requests:
      storage: 2Gi
  storageClassName: hostpath

podAntiAffinity: {}

3) kubectl create namespace elastic-stack
4) kubectl config set-context --current --namespace=elastic-stack 
5) helm install elasticsearch elastic/elasticsearch -f es-values.yaml --namespace=elastic-stack

#filebeat.yaml

  enabled: true

deployment:
  enabled: false

filebeatConfig:
  filebeat.yml: |
    filebeat.inputs:
      - type: container
        enabled: true
        paths:
          - /var/log/containers/*.log
        processors:
          - add_kubernetes_metadata:
              host: ${NODE_NAME}
              matchers:
                - logs_path:
                    logs_path: "/var/log/containers/"

    output.elasticsearch:
      hosts: ["https://elasticsearch-master:9200"]
      username: "elastic"
      password: "Y5DjZgQbK8GIyGfu"
      ssl:
        certificate_authorities: ["/usr/share/filebeat/elastic-ca.crt"]

extraVolumes:
  - name: elastic-ca
    secret:
      secretName: elasticsearch-master-certs
      items:
        - key: ca.crt
          path: elastic-ca.crt

extraVolumeMounts:
  - name: elastic-ca
    mountPath: /usr/share/filebeat/elastic-ca.crt
    subPath: elastic-ca.crt

resources:
  requests:
    cpu: 100m
    memory: 200Mi
  limits:
    cpu: 200m
    memory: 300Mi

#pod.yaml

apiVersion: v1
kind: Pod
metadata:
  name: log-generator
  namespace: elastic-stack
  labels:
    app: log-generator
spec:
  containers:
  - name: logger
    image: busybox
    command: ["/bin/sh"]
    args:
      - "-c"
      - "while true; do echo \"Hello From Pod $(date)\"; sleep 1; done"

18) kubectl apply -f pod.yaml
19) kubectl logs log-generator --namespace=elastic-stack -f 
20) kubectl port-forward service/kibana-kibana 5601:5601 -n elastic-stack
21) Create a Data View: Kibana → Stack Management → Data Views
Data Views (Name: filebeat-*, Value: filebeat-*, time field: @timestamp)
Check in Discover: Discover → choose our filebeat-* data view.

eg KQL:
message : "Hello From Pod"
kubernetes.namespace : "elastic-stack"
kubernetes.pod.name : "elasticsearch-master-0"

Note: Change the time range to get real time values

Your setup flows from Pod → Filebeat → Elasticsearch → Kibana
