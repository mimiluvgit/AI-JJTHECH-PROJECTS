#### What is ARGO CD ####
Argo CD is a GitOps continuous delivery tool for Kubernetes as it automatically deploys the state of your apps defined 
in Git (so Git is your “source of truth”)

It’s a declarative, GitOps tool for Kubernetes.
You store your Kubernetes manifests configs in Git.
Argo CD automatically syncs your cluster state with what’s in Git

#### Step 1: Create Namespace and Install Argo CD ####

kubectl create namespace argocd
kubectl apply -n argocd \
  -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

Pod					Purpose
argocd-application-controller		Continuously monitors Git & syncs apps to match desired state in the cluster.
argocd-applicationset-controller	Automates creation of multiple Argo CD Applications from templates 
argocd-dex-server			Handles authentication and Single Sign-On (SSO) for the Argo CD UI and API.
argocd-notifications-controller		Sends notifications (Slack, email, etc.) when app events occur.
argocd-redis				In-memory database that caches app and repo metadata to speed up Argo CD operations.
argocd-repo-server			Fetches Git repos and generates Kubernetes manifests for the controller.
argocd-server				Exposes the Argo CD API and Web UI for user access.

kubectl patch svc argocd-server -n argocd -p '{"spec": {"type": "NodePort"}}'
kubectl get svc argocd-server -n argocd
echo "Username: admin" && kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d; 
open https://localhost:<NODE_PORT>

#### Create Your own Git Repository ###

Make a repo that contains your Kubernetes manifests (example k8s/deployment.yaml, k8s/service.yaml).
Push it to GitHub or GitLab — this is your source of truth.

eg: Refer: https://github.com/sunnyaws1984/my-argo-demo.git

#### ARGO CD configuration #####

kubectl create namespace dev

kubectl apply -f argo-app.yaml
** prune: true     -> Delete (prune) resources that are no longer defined in Git
** selfHeal: true  -> Detect and fix manual changes made directly in the cluster

kubectl get Application -n argocd
**SYNC STATUS: Synced → means your cluster’s live state matches what’s defined in your Git repo.
**HEALTH STATUS: Healthy → means all resources (pods, services, etc.) are up and running.

kubectl get pods -n dev

##### Verification ######

-- Change the replica to 2 in repo
